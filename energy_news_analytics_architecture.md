## 1. 系统总体架构

### 1.1 架构风格选择
**推荐架构: 微服务架构 + 前后端分离**

选择理由：
1. **多维度分析独立性**: 基本面、情绪面、技术面等分析可独立部署和扩展
2. **品种可扩展性**: 新增能源品种只需添加对应的数据采集和分析模块
3. **技术栈灵活性**: 不同分析模块可采用最适合的技术（如NLP用Python，实时计算用Go）
4. **故障隔离**: 单个分析模块故障不影响整体系统
5. **团队并行开发**: 不同团队可独立开发和部署各自服务

## 2. 核心模块定义

### 2.1 数据采集模块 (Data Collection Service)

**职责**: 负责从多渠道获取新闻数据

**子模块**:
| 子模块 | 功能描述 | 技术实现 |
|--------|----------|----------|
| 爬虫引擎 | 自动抓取专业媒体和主流网站 | Scrapy + Playwright |
| RSS聚合器 | 订阅RSS源获取新闻 | Feedparser |
| API接入 | 对接第三方数据提供商 | REST API Client |
| 手工导入 | 处理用户上传的URL/文本/PDF | REST API + 文件处理 |
| PDF解析 | 提取PDF中的文本内容 | PyPDF2 / pdfplumber |

**支持的信源类型**:
- 专业媒体: Platts, Argus, ICIS, 隆众资讯, 金联创
- 主流媒体: 新华社, 路透社, 彭博, 华尔街日报
- 官方渠道: EIA, OPEC, IEA, 各国能源部门
- 社交平台: Twitter/X, 雪球, 东方财富

**关注品种配置**:
```yaml
commodities:
  crude_oil:
    - WTI
    - Brent
  natural_gas:
    - JKM
    - HH
    - TTF
  lpg:
    - PG
    - PP
    - MB
    - FEI
    - CP
  oil_products:
    - gasoline
    - diesel
    - fuel_oil
```

### 2.2 数据预处理模块 (Data Preprocessing Service)

**职责**: 清洗、标准化和结构化原始数据

**核心功能**:
1. **文本清洗**: 去除HTML标签、广告、重复内容
2. **去重处理**: 基于SimHash/MinHash的相似度去重
3. **实体识别**: 识别能源品种、地区、公司、人物
4. **时间标准化**: 统一时间格式，提取发布时间
5. **分类标签**: 自动打标签（品种、地区、事件类型）

**处理流程**:
```
原始数据 → 格式检测 → 内容提取 → 去重判断 → 实体识别 → 标签标注 → 存储
```

### 2.3 多维度分析引擎 (Multi-dimensional Analysis Engine)

#### 2.3.1 基本面分析服务 (Fundamental Analysis Service)

**职责**: 分析供需、库存、产量等基本面数据

**分析维度**:
| 维度 | 分析内容 | 输出指标 |
|------|----------|----------|
| 供应分析 | OPEC+产量、美国页岩油、俄罗斯出口 | 供应变化率、预期差 |
| 需求分析 | 全球GDP、炼厂开工率、季节性需求 | 需求增长率 |
| 库存分析 | EIA/API库存、战略储备、浮仓 | 库存变化、库销比 |
| 价差分析 | 裂解价差、跨区价差、月差 | 价差走势、套利机会 |

**技术实现**:
- NLP模型提取供需数据
- 时间序列分析库存趋势
- 规则引擎判断供需平衡状态

#### 2.3.2 宏观面分析服务 (Macro Analysis Service)

**职责**: 分析宏观经济对能源市场的影响

**分析维度**:
| 维度 | 分析内容 | 输出指标 |
|------|----------|----------|
| 货币政策 | 美联储利率、美元指数 | 美元影响系数 |
| 经济数据 | GDP、CPI、PMI、就业数据 | 经济景气指数 |
| 汇率影响 | 主要货币汇率波动 | 汇率敏感度 |
| 全球贸易 | 贸易政策、关税变化 | 贸易影响评估 |

#### 2.3.3 情绪面分析服务 (Sentiment Analysis Service)

**职责**: 分析市场情绪和市场参与者心理

**分析维度**:
| 维度 | 分析内容 | 输出指标 |
|------|----------|----------|
| 文本情绪 | 新闻情感极性、情感强度 | 情绪得分(-1~1) |
| 市场情绪 | 投机情绪、恐慌指数 | 市场情绪指数 |
| 机构观点 | 投行研报、分析师评级 | 机构一致性指数 |
| 社交媒体 | 讨论热度、话题趋势 | 热度指数 |

**技术实现**:
- 预训练语言模型: FinBERT / RoBERTa
- 情感分类: 正面/负面/中性
- 情绪强度: 回归模型输出0-1分值
- 多语言支持: 中英文混合分析

#### 2.3.4 技术面分析服务 (Technical Analysis Service)

**职责**: 分析价格走势和技术指标

**分析维度**:
| 维度 | 分析内容 | 输出指标 |
|------|----------|----------|
| 趋势分析 | 均线排列、趋势线 | 趋势方向/强度 |
| 形态识别 | 头肩顶/底、双顶/底 | 形态信号 |
| 技术指标 | RSI、MACD、布林带 | 指标数值/信号 |
| 支撑阻力 | 关键价位识别 | 支撑/阻力位 |

**技术实现**:
- TA-Lib技术指标库
- 实时价格数据接入
- 形态识别算法

#### 2.3.5 地缘风险分析服务 (Geopolitical Risk Service)

**职责**: 分析地缘政治事件对能源市场的影响

**分析维度**:
| 维度 | 分析内容 | 输出指标 |
|------|----------|----------|
| 地区冲突 | 中东局势、俄乌冲突 | 风险等级(1-5) |
| 制裁影响 | 对俄制裁、伊朗制裁 | 供应影响评估 |
| 政策风险 | 能源政策、环保法规 | 政策影响指数 |
| 运输风险 | 海峡通行、管道安全 | 运输风险指数 |

**技术实现**:
- 地缘政治知识图谱
- 事件影响模型
- 历史案例匹配

### 2.4 量化分析服务 (Quantitative Analysis Service)

**职责**: 整合多维度分析结果，生成量化指标

**核心功能**:
1. **多因子模型**: 综合各维度因子生成综合评分
2. **权重配置**: 支持用户自定义各维度权重
3. **信号生成**: 生成交易/配置建议信号
4. **回测验证**: 历史信号效果回测
5. **风险预警**: 异常波动和风险事件预警

**量化指标输出**:
```python
{
  "commodity": "WTI",
  "timestamp": "2025-01-15T10:00:00Z",
  "composite_score": 0.65,  # 综合评分 -1~1
  "dimension_scores": {
    "fundamental": 0.45,
    "macro": 0.30,
    "sentiment": 0.75,
    "technical": 0.55,
    "geopolitical": -0.20
  },
  "signal": "CAUTIOUS_BULLISH",  # 谨慎看多
  "confidence": 0.72,  # 置信度
  "risk_level": "MEDIUM",
  "key_factors": ["库存下降", "地缘紧张", "美元走弱"]
}
```

## 3. 模块间接口设计

### 3.1 接口规范

**通信协议**: REST API + gRPC + 消息队列

**数据格式**: JSON / Protocol Buffers

**认证方式**: JWT Token

### 3.2 核心API定义

#### 数据采集接口
```yaml
# 手工导入新闻
POST /api/v1/news/import
Request:
  type: url | text | pdf
  content: string | file
  source: string
  commodity: string[]
  
Response:
  id: string
  status: processing | completed | failed
  message: string

# 获取新闻列表
GET /api/v1/news
Parameters:
  commodity: string
  start_date: date
  end_date: date
  source: string
  sentiment: positive | negative | neutral
  
Response:
  items: NewsItem[]
  total: number
  page: number
```

#### 分析接口
```yaml
# 获取多维度分析结果
GET /api/v1/analysis/{commodity}
Parameters:
  dimensions: fundamental,macro,sentiment,technical,geopolitical
  
Response:
  commodity: string
  timestamp: datetime
  dimensions:
    fundamental:
      score: float
      factors: Factor[]
    sentiment:
      score: float
      distribution: {positive: float, negative: float, neutral: float}
    # ... 其他维度
  composite_score: float
  signal: string

# 获取情绪分析详情
GET /api/v1/sentiment/{commodity}
Response:
  overall_score: float
  trend: rising | falling | stable
  news_sentiment: NewsSentiment[]
  social_sentiment: SocialSentiment
  historical: TimeSeries[]
```

#### 量化接口
```yaml
# 获取量化评分
GET /api/v1/quant/{commodity}
Response:
  score: float
  signal: BULLISH | BEARISH | NEUTRAL
  confidence: float
  factors: Factor[]
  backtest_result: BacktestResult

# 获取预警列表
GET /api/v1/alerts
Response:
  alerts: Alert[]
  unread_count: number
```

### 3.3 消息队列设计

**Kafka Topic设计**:
```
# 原始数据流
raw.news.{source}          # 原始新闻数据
raw.price.{commodity}      # 原始价格数据

# 处理数据流
processed.news             # 清洗后的新闻
processed.sentiment        # 情绪分析结果
processed.fundamental      # 基本面分析结果

# 聚合数据流
aggregated.daily           # 日度聚合数据
aggregated.signals         # 交易信号

# 系统事件流
system.alerts              # 系统预警
system.reports             # 报告生成事件
```

   - 读写分离
   - 分库分表（按时间/品种）
   - 索引优化
   - 查询优化

   - 非关键路径异步化
   - 消息队列削峰填谷

4. **计算优化**:
   - NLP模型GPU加速
   - 向量检索近似算法

## 12. 总结

本架构设计采用微服务架构，具有以下特点：

1. **高可扩展性**: 各分析维度独立部署，新功能可快速添加
2. **高性能**: 异步处理 + 缓存 + 消息队列，确保高并发处理能力
3. **高可用性**: 多节点部署 + 故障隔离，确保系统稳定运行
4. **易维护性**: 模块化设计 + 完善的监控体系，降低运维成本
5. **技术先进**: 采用云原生技术栈，支持容器化部署和自动化运维
